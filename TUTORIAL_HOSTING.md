# Tutorial Menjalankan WebGIS Drainase di Hosting (Ubuntu VPS)\n\n## 📋 Persiapan Awal\n\n### 1. Requirements Hosting\n- **OS**: Ubuntu 20.04 LTS atau lebih baru\n- **RAM**: Minimum 2GB (recommended 4GB)\n- **Storage**: Minimum 20GB\n- **Domain**: Siapkan domain sendiri atau gunakan subdomain\n- **Root/Sudo Access**: Diperlukan untuk setup\n\n### 2. Akses Hosting via SSH\n\n```bash\n# Dari terminal lokal, masuk ke VPS\nssh root@your_server_ip\n\n# Jika menggunakan SSH key\nssh -i /path/to/key.pem root@your_server_ip\n```\n\n---\n\n## 🔧 Step-by-Step Installation\n\n### Step 1: Update Server\n\n```bash\n# Update package manager\nsudo apt update && sudo apt upgrade -y\n\n# Install dependencies dasar\nsudo apt install -y curl wget git nano htop\n```\n\n### Step 2: Install Docker & Docker Compose\n\n```bash\n# Install Docker\ncurl -fsSL https://get.docker.com -o get-docker.sh\nsudo sh get-docker.sh\n\n# Verifikasi Docker\ndocker --version\n\n# Install Docker Compose\nsudo curl -L \"https://github.com/docker/compose/releases/download/v2.20.0/docker-compose-$(uname -s)-$(uname -m)\" -o /usr/local/bin/docker-compose\nsudo chmod +x /usr/local/bin/docker-compose\n\n# Verifikasi Docker Compose\ndocker-compose --version\n\n# Tambahkan user ke docker group (opsional)\nsudo usermod -aG docker $USER\nnewgrp docker\n```\n\n### Step 3: Setup Directory Aplikasi\n\n```bash\n# Buat direktori aplikasi\nsudo mkdir -p /var/www\ncd /var/www\n\n# Clone repository\nsudo git clone https://github.com/51p3/webgis-drainase.git\ncd webgis-drainase\n\n# Ubah permissions\nsudo chown -R $USER:$USER /var/www/webgis-drainase\n```\n\n### Step 4: Konfigurasi Environment\n\n```bash\n# Copy file environment\ncp .env.example .env\n\n# Edit dengan nano\nnano .env\n```\n\n**Edit nilai berikut di file .env:**\n\n```bash\n# Aplikasi\nAPP_NAME=\"WebGIS Drainase Brebes\"\nAPP_ENV=production\nAPP_DEBUG=false\nAPP_URL=https://yourdomain.com\n\n# Database\nDB_CONNECTION=pgsql\nDB_HOST=postgres\nDB_PORT=5432\nDB_DATABASE=webgis_drainase\nDB_USERNAME=webgis_user\nDB_PASSWORD=GANTI_DENGAN_PASSWORD_KUAT_12345\n\n# Frontend\nVITE_API_URL=https://yourdomain.com/api\n\n# Map Configuration (Koordinat Brebes)\nMAP_DEFAULT_CENTER_LAT=-7.2575\nMAP_DEFAULT_CENTER_LNG=108.7392\nMAP_DEFAULT_ZOOM=10\n```\n\n**Simpan dengan:**\n- Tekan `Ctrl + X`\n- Tekan `Y` (Yes)\n- Tekan `Enter`\n\n### Step 5: Setup SSL Certificate (Let's Encrypt)\n\n```bash\n# Install Certbot\nsudo apt install -y certbot python3-certbot-nginx\n\n# Generate SSL Certificate\n# CATATAN: Pastikan domain sudah pointing ke IP VPS\nsudo certbot certonly --standalone -d yourdomain.com -d www.yourdomain.com\n\n# Akan diminta email, setuju terms, dll\n# Certificate disimpan di: /etc/letsencrypt/live/yourdomain.com/\n\n# Copy ke folder aplikasi\nsudo mkdir -p /var/www/webgis-drainase/nginx/ssl\nsudo cp /etc/letsencrypt/live/yourdomain.com/fullchain.pem /var/www/webgis-drainase/nginx/ssl/\nsudo cp /etc/letsencrypt/live/yourdomain.com/privkey.pem /var/www/webgis-drainase/nginx/ssl/\nsudo chown -R $USER:$USER /var/www/webgis-drainase/nginx/ssl\n```\n\n### Step 6: Update Nginx Configuration\n\n```bash\n# Edit file nginx config\nnano /var/www/webgis-drainase/nginx/conf.d/default.conf\n```\n\n**Ganti `yourdomain.com` dengan domain Anda:**\n\n```bash\nserver {\n    listen 80;\n    server_name yourdomain.com www.yourdomain.com;\n    return 301 https://$server_name$request_uri;\n}\n\nserver {\n    listen 443 ssl http2;\n    server_name yourdomain.com www.yourdomain.com;\n\n    ssl_certificate /etc/nginx/ssl/fullchain.pem;\n    ssl_certificate_key /etc/nginx/ssl/privkey.pem;\n    ssl_protocols TLSv1.2 TLSv1.3;\n    ssl_ciphers HIGH:!aNULL:!MD5;\n\n    root /var/www/html;\n    index index.html;\n\n    location / {\n        try_files $uri $uri/ /index.html;\n    }\n\n    location /api/ {\n        proxy_pass http://backend:8000/api/;\n        proxy_set_header Host $host;\n        proxy_set_header X-Real-IP $remote_addr;\n        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;\n        proxy_set_header X-Forwarded-Proto $scheme;\n    }\n\n    location ~* \\.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {\n        expires 1y;\n        add_header Cache-Control \"public, immutable\";\n    }\n}\n```\n\n### Step 7: Start Docker Containers\n\n```bash\n# Navigate ke folder aplikasi\ncd /var/www/webgis-drainase\n\n# Build dan jalankan container production\ndocker-compose -f docker-compose.prod.yml up -d\n\n# Tunggu 30 detik untuk container startup\nsleep 30\n\n# Cek status container\ndocker-compose -f docker-compose.prod.yml ps\n\n# Lihat logs (jika ada error)\ndocker-compose -f docker-compose.prod.yml logs -f\n```\n\n### Step 8: Database Migration\n\n```bash\n# Tunggu database siap (sekitar 30 detik)\nsleep 30\n\n# Generate application key\ndocker-compose -f docker-compose.prod.yml exec backend php artisan key:generate\n\n# Run database migrations\ndocker-compose -f docker-compose.prod.yml exec backend php artisan migrate --force\n\n# Seed database dengan data sample\ndocker-compose -f docker-compose.prod.yml exec backend php artisan db:seed\n\n# Clear cache\ndocker-compose -f docker-compose.prod.yml exec backend php artisan cache:clear\ndocker-compose -f docker-compose.prod.yml exec backend php artisan config:cache\n```\n\n### Step 9: Konfigurasi Domain (DNS)\n\n**Jika belum setup domain:**\n\n1. Buka DNS provider Anda (Cloudflare, Niagahoster, etc)\n2. Tambah A record:\n   - **Name**: @ (atau yourdomain.com)\n   - **Type**: A\n   - **Value**: IP VPS Anda\n3. Tambah CNAME record:\n   - **Name**: www\n   - **Type**: CNAME\n   - **Value**: yourdomain.com\n\n**Tunggu 5-30 menit untuk DNS propagation**\n\n### Step 10: Verifikasi Instalasi\n\n```bash\n# Check API\ncurl https://yourdomain.com/api\n\n# Cek status container\ndocker-compose -f docker-compose.prod.yml ps\n\n# Test database connection\ndocker-compose -f docker-compose.prod.yml exec backend php artisan tinker\n# Di tinker shell, ketik: DB::connection()->getPdo();\n# Exit dengan: exit\n```\n\n---\n\n## 🌐 Akses Aplikasi\n\n```\nURL Frontend: https://yourdomain.com\nURL Backend API: Hash::make('password_baru_yang_kuat');\n$user->save();\nexit\n```\n\n---\n\n## 📊 Maintenance & Monitoring\n\n### Melihat Logs\n\n```bash\n# Backend logs\ndocker-compose -f docker-compose.prod.yml logs -f backend\n\n# Frontend logs\ndocker-compose -f docker-compose.prod.yml logs -f frontend\n\n# Nginx logs\ndocker-compose -f docker-compose.prod.yml logs -f nginx\n\n# Database logs\ndocker-compose -f docker-compose.prod.yml logs -f postgres\n```\n\n### Restart Container\n\n```bash\n# Restart semua container\ndocker-compose -f docker-compose.prod.yml restart\n\n# Restart container tertentu\ndocker-compose -f docker-compose.prod.yml restart backend\n```\n\n### Update Aplikasi\n\n```bash\ncd /var/www/webgis-drainase\n\n# Pull latest code\ngit pull origin main\n\n# Rebuild containers\ndocker-compose -f docker-compose.prod.yml up -d --build\n\n# Run migrations (jika ada)\ndocker-compose -f docker-compose.prod.yml exec backend php artisan migrate --force\n```\n\n### Backup Database\n\n```bash\n# Backup ke file SQL\ndocker-compose -f docker-compose.prod.yml exec -T postgres pg_dump -U webgis_user webgis_drainase > backup_$(date +%Y%m%d_%H%M%S).sql\n\n# Gzip compress\ngzip backup_*.sql\n\n# Download ke local\nscp root@your_server_ip:/var/www/webgis-drainase/backup_*.sql.gz .\n```\n\n### Auto Backup dengan Cron\n\n```bash\n# Edit crontab\ncrontab -e\n\n# Tambahkan line ini (backup setiap hari jam 2 pagi):\n0 2 * * * cd /var/www/webgis-drainase && docker-compose -f docker-compose.prod.yml exec -T postgres pg_dump -U webgis_user webgis_drainase | gzip > /backups/db_$(date +\\%Y\\%m\\%d_\\%H\\%M\\%S).sql.gz\n\n# Buat folder backups terlebih dahulu\nsudo mkdir -p /backups\nsudo chown $USER:$USER /backups\n```\n\n---\n\n## 🐛 Troubleshooting\n\n### Problem: Container tidak jalan\n\n```bash\n# Cek logs\ndocker-compose -f docker-compose.prod.yml logs backend\n\n# Restart container\ndocker-compose -f docker-compose.prod.yml restart\n\n# Rebuild container\ndocker-compose -f docker-compose.prod.yml up -d --build\n```\n\n### Problem: Database connection error\n\n```bash\n# Cek apakah postgres container running\ndocker-compose -f docker-compose.prod.yml ps postgres\n\n# Tunggu 30 detik dan restart\nsleep 30\ndocker-compose -f docker-compose.prod.yml restart postgres\n\n# Cek logs postgres\ndocker-compose -f docker-compose.prod.yml logs postgres\n```\n\n### Problem: API tidak respond\n\n```bash\n# Clear laravel cache\ndocker-compose -f docker-compose.prod.yml exec backend php artisan cache:clear\ndocker-compose -f docker-compose.prod.yml exec backend php artisan config:cache\n\n# Restart backend\ndocker-compose -f docker-compose.prod.yml restart backend\n```\n\n### Problem: Frontend blank/white page\n\n```bash\n# Cek browser console (F12)\n# Lihat error di network tab\n\n# Pastikan VITE_API_URL di .env sudah benar\n# Rebuild frontend\ndocker-compose -f docker-compose.prod.yml up -d --build frontend\n\n# Clear browser cache (Ctrl+Shift+Delete)\n```\n\n### Problem: SSL Certificate error\n\n```bash\n# Renew certificate\nsudo certbot renew --dry-run\nsudo certbot renew\n\n# Copy ke aplikasi\nsudo cp /etc/letsencrypt/live/yourdomain.com/fullchain.pem /var/www/webgis-drainase/nginx/ssl/\nsudo cp /etc/letsencrypt/live/yourdomain.com/privkey.pem /var/www/webgis-drainase/nginx/ssl/\n\n# Restart nginx\ndocker-compose -f docker-compose.prod.yml restart nginx\n```\n\n---\n\n## 📱 Testing
